PocketHusky

Group Project

My Role

Developer

Writer

A fun mobile app to encourage students to explore and learn about what is on the UW Bothell campus

Visit website

Purpose

Pocket Husky was a group project that spanned two quarters starting in winter quarter of 2016. The class was designed to be more like a studio class and free reign was given over what students wanted to work on, given that it related to a social issue. My group decided on student engagement and awareness on campus as the main focus for our project.

Development

From the idea of student awareness we listed off ideas on what the project itself would be, eventually we agreed upon the goal of developing an app that would encourage students to walk around campus and inform them about the services available. I was in charge of creating an overlay for the Google Maps API so it would have more clarity and cohesiveness than satellite imagery, and making sure we could access the user's coordinates while the app was open. Accessing the phone's coordinates was simple, and I discovered that it could detect the elevation, however, there was a wide margin of accuracy which meant the group needed to rethink the aspects of the app that relied on precise location information. Figuring out how to overlay images on top of the Google Map satellite image took a majority of development time, mainly due to the fact that I was still new to JavaScript and coding in general. Once all the images were properly aligned I enabled the viewing of the different floors by disabling the overlays of all the other floors except whichever was currently selected.

Outcome

At the start of this project I only had a quarter's worth of web development knowledge, which only introduced JavaScript near the end. This meant that I had to learn alongside reading the Google Maps API to get the desired outcome, it was a combination of having difficulty understanding the API documentation due to my lack of JavaScript understanding, as well as the API just not being able to perform the task I wanted it to. Although, it was very encouraging to have it work in the end and have everyone's hard work pay off, even if not everything we wanted was included in the final version. This experience helped show me how much JavaScript could do and that I needed to keep working with it in order to understand and get better at it.
